### 2026-05-16
- 문서 운영 규칙 준수 및 무한 반복 설정
  - 개발 내용은 `Development.md`, 실수/개선점은 `AGENTS.md`에 기록하는 정책을 수용함.
  - 모든 작업 완료 시 해당 문서들을 우선적으로 업데이트하는 프로세스를 확립함.
- 아티팩트 저장 경로 오류 수정
  - **실수**: `write_to_file` 시 아티팩트를 프로젝트 루트에 저장하려 함.
  - **원인**: 아티팩트 전용 경로(`appDataDir/brain/conv-id/`) 규칙을 일시적으로 망각함.
  - **재발 방지**: 아티팩트 생성 시 반드시 지정된 App Data Directory 경로를 확인하고 사용함.
- **성공적 기능 구현**:
  - DB 없이도 실시간 데이터 가공을 통해 '시장 감성 지수'를 구현하여 시스템 복잡도를 낮춤.
  - `localStorage`를 활용해 사용자 맞춤형 기능을 빠르게 제공함.
- **모듈화 및 테스트 최적화 성공**:
  - 부수 효과(Side Effects)가 없는 순수 함수 위주로 로직을 재배치하여 테스트 코드를 즉시 작성할 수 있는 구조로 개선함.
  - 비즈니스 로직을 UI로부터 완전히 격리하여 향후 기능 확장 시 안전한 변경이 가능해짐.
- **UI 컴포넌트 커버리지 100% 달성**:
  - `jsdom`과 `React Testing Library`를 활용하여 모든 페이지 및 컴포넌트의 렌더링 상태를 전수 검증함.
  - 비즈니스 로직(100%)과 UI(100%)가 모두 테스트로 보호되는 고품질 코드베이스를 구축함.
- **실시간 체결강도 스캐너(KIS API) 통합 성공**:
  - 한국투자증권 API 연동을 통해 100% 실시간 데이터 기반의 '체결강도 TOP 10' 스캐너를 구현함.
  - API 키 부재 시에도 UI가 작동하도록 Mock 데이터 폴백 로직을 설계하여 안정성을 확보함.
- **OpenDART API 심층 연동 패턴 구축**:
  - 무거운 `CORPCODE.xml`을 다운로드하는 대신, `list.json`을 특정 일자로 쿼리하여 해당 공시의 `corp_code`를 동적으로 알아내는 가벼운(Serverless-friendly) 방식을 고안하여 성공적으로 적용함.
  - 이를 통해 가장 시장 파급력이 큰 "단일판매ㆍ공급계약체결" 공시의 실제 금액과 비율을 UI에 즉시 노출할 수 있게 됨.
- **빌드 시스템 안정화**:
  - 리팩토링 중 누락된 변수 참조로 인한 Netlify 빌드 실패를 즉시 감지하고 복구함.
  - 그리드 레이아웃을 반응형 플렉스 레이아웃으로 개선하여 신규 컴포넌트 추가 시의 레이아웃 깨짐 현상을 방지함.
  - `Next.js`의 프로덕션 빌드 단계에서 `vitest` 관련 타입 선언 에러(`Cannot find module 'vitest/config'`)가 발생한 것을 파악하고, `tsconfig.json`의 `exclude` 속성에 테스트 관련 파일(`**/*.test.ts`, `vitest.config.ts` 등)을 추가하여 빌드 프로세스를 완벽히 안정화함.
