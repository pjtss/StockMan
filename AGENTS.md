### 2026-05-17
- **아이폰 12 Pro(390px) GNB 네비게이션 및 홈 화면 액션 버튼 우측 잘림 철저 차단**:
  - **실수**: GNB(PageNavigation)의 가로 스크롤 레이아웃과 홈 화면 메인 히어로 카드의 액션 버튼 가로 배치가 좁은 모바일 화면(390px 이하)에서 flex container 오버플로우 한계를 넘겨 우측 레이아웃을 삐져나가게 만드는 엣지 케이스를 미처 정밀하게 수정하지 못함.
  - **원인**: GNB `.nav`에 지정된 `max-content: 100%` 오타(Syntax Error)와 flexbox의 align-items 속성으로 인해 실제 너비 한계가 풀려 스크롤이 무력화되었고, 홈 화면의 액션 버튼들은 좁은 여백 속에서 wrap될 때도 글자수가 너무 길어 가로를 침범함.
  - **해결 및 재발 방지**: 
    1. GNB `.nav` 오타를 `max-width: 100%`로 완벽 복구하고, 모바일 미디어 쿼리 내에 `max-width: 100%`를 한 번 더 단단히 선언하여 가로 뷰포트 바깥으로 절대 삐져나가지 않는 안전한 가로 스크롤(Horizontal Swipe) 메커니즘을 확실히 보장함.
    2. 홈 화면의 액션 버튼들은 모바일 미디어 쿼리(`max-width: 480px`) 하위에서 가로형 flex를 **세로형 블록 스택 구조(`flex-direction: column; width: 100%;`)**로 완전 개편하고 히어로 카드 내부 패딩을 `16px`로 정밀 축소하여, 아이폰 12 Pro를 포함한 전 기종의 모바일 브라우저에서 단 1픽셀의 잘림 현상도 없이 100% 완전무결하게 정비 완료함.
- **Next.js 자동 preload 브라우저 경고(not used within a few seconds) 완전 퇴치**:
  - **실수**: Next.js의 `<Link>` 컴포넌트가 브라우저 뷰포트 내의 모든 링크 대상을 자동으로 사전 로드(Prefetch)하도록 기본 설정해 두었으나, 사용자가 즉시 해당 페이지로 이동하지 않을 경우 브라우저 콘솔에 preloaded but not used 경고가 다수 유출되는 현상을 세밀하게 대응하지 못함.
  - **원인**: Next.js App Router의 기본 프리페치 정책에 따라 사용자가 첫 화면(Home)을 보고 있을 때 헤더 GNB의 모든 링크가 사전 로드되면서, 로드 이벤트 후 수 초 내에 사용되지 않아 브라우저가 네트워크 및 메모리 낭비 경고를 발생시킴.
  - **해결 및 재발 방지**: 헤더 내비게이션 및 홈 화면의 모든 `<Link>` 컴포넌트에 `prefetch={false}` 속성을 명시적으로 부여. 이로써 불필요한 초기 데이터 백그라운드 사전 로드를 차단해 모바일 대역폭과 성능을 획기적으로 절약하고, 브라우저의 preload 미사용 경고를 완벽하게 영구 퇴치 완료. 사용자가 링크에 마우스를 올리거나(Hover) 상호작용할 때만 동적으로 프리페치가 수행되므로 instant 이동 성능은 여전히 완벽하게 유지됨.
- **아이폰 12 Pro(390px) 좁은 뷰포트 수급 스캐너 우측 잘림 근본 차단**:
  - **실수**: 홈 화면과 DART 공시 페이지의 메이저 수급 스캐너 탭은 스와이프 flex 레이아웃으로 변경하였으나, 탭 내부에 출력되는 종목별 리스트 `.row` 영역의 텍스트가 너무 길어(`외인 +2,159억 | 기관 -124억` 등) 아이폰 12 Pro(390px)와 같은 좁은 뷰포트에서 오른쪽으로 삐져나가 가로 overflow 잘림을 일으키는 현상을 간과함.
  - **원인**: flex 1열 행 배치 방식으로는 수급 매매 텍스트의 총 가로 점유 너비가 뷰포트 영역을 강제로 침범하여 clipping 또는 page horizontal overflow를 일으킬 수밖에 없었음.
  - **해결 및 재발 방지**: 모바일 전용 미디어 쿼리(`max-width: 480px`) 내에서 `.row`를 2열 grid가 아닌 **2행 3열 grid 구조(`grid-template-rows: auto auto`)**로 정비. 1행에는 순위, 회사명, 가격을 두고, 2행의 전체 가로 영역(`grid-column: 2 / -1`)을 수급 텍스트 `.metricCol`이 독점해서 차지하게 함으로써, 텍스트가 아무리 길어도 아이폰 12 Pro와 초소형 화면(iPhone SE 등)에서 가로 잘림 없이 100% 자가 적응하도록 개정 완료.
