# Project Improvement & Feature Design Plan

이 문서는 현재 RSS Dashboard 프로젝트의 분석과 향후 개선 및 추가 기능 설계를 담고 있습니다.

## 1. 현재 프로젝트 분석 (Current State)

### 기술 스택
- **Framework**: Next.js 15 (App Router)
- **Language**: TypeScript
- **Feed Source**: DART (RSS), SEC (Atom/RSS)
- **Notifications**: Web Push (web-push)
- **Database**: PostgreSQL (pg) - *현재 비활성화(Stateless 모드)*
- **Styling**: CSS Modules

### 주요 기능
- DART 및 SEC 공시 실시간 크롤링 (Polling 방식)
- 키워드 기반의 감성 분석 (호재, 악재, 중립 등)
- 브라우저 푸시 알림 (최근 1분 내 공시 대상)
- 실시간 대시보드 (Rapid Dart Page)

## 2. 제안하는 개선점 및 신규 기능 (Proposed Features)

### 🎨 A. 프리미엄 UI/UX 고도화 (Modern Design System)
- **개요**: 현재의 정적인 디자인을 더 생동감 있고 프리미엄한 느낌으로 개선.
- **상세 설계**:
    - **Glassmorphism**: 카드 디자인에 투명도와 블러 효과 적용.
    - **Micro-animations**: 새로운 공시가 올라올 때 부드러운 페이드인 및 하이라이트 효과.
    - **Chart Integration**: 종목명 클릭 시 네이버 증권 또는 TradingView 차트 팝업 연동.

### 📊 B. 키워드 로직 고도화 및 정밀 분류
- **개요**: 단순 키워드 포함 여부가 아닌, 가중치와 맥락을 고려한 정밀 분류 시스템 구축.
- **상세 설계**:
    - **Scoring System**: 키워드별 점수 부여 및 합산 방식 도입.
    - **Regex Matching**: 정교한 정규표현식을 통한 문구 매칭.
    - **Exclusion Logic**: 호재 키워드가 있어도 '해지', '철회' 등이 포함된 경우 자동 악재 분류.

### 📊 C. OpenDART API 심층 연동
- **개요**: RSS가 제공하는 제한된 정보(제목, 링크)를 넘어 실제 공시 본문 데이터 활용.
- **상세 설계**:
    - OpenDART API를 통해 기업의 재무 정보나 구체적인 계약 금액 등을 추출하여 대시보드에 직접 표시.

## 5. 상세 설계: Phase 2 - 키워드 로직 고도화

현재의 단순 키워드 매칭 방식(`includes`)에서 발생하는 오판을 줄이고, 공시의 맥락을 더 정확히 파악하기 위한 고도화 설계입니다.

### 5.1 복합 키워드 및 가중치 시스템 (Scoring System)
- **개요**: 단일 키워드가 아닌 여러 키워드의 조합에 따라 점수를 부여하여 신뢰도를 높임.
- **상세**:
    - **가중치 부여**: 각 키워드에 점수(Weight) 할당. (예: `공급계약`: +10, `대규모`: +5, `정정`: -3)
    - **임계값 설정**: 합산 점수가 특정 점수 이상일 때만 '최강호재'로 분류.
    - **시너지 효과**: 특정 키워드 쌍이 동시 발견될 경우 보너스 점수 부여. (예: `영업이익` + `흑자전환`)

### 5.2 맥락 기반 예외 처리 (Context-Aware Exclusion)
- **개요**: 호재 키워드가 포함되어 있어도 특정 상황에서는 중립 또는 악재로 분류.
- **상세**:
    - **정정 공시 처리**: `[정정]` 키워드 발견 시, 이전 공시와의 변경 사항을 체크하도록 로직 분기. (금액 감소 시 악재로 분류)
    - **상대적 규모 파악**: '매출액 대비 50% 이상'과 같은 수치적 표현 패턴 매칭 (Regex 활용).
    - **제3자 배정 유상증자**: 일반 유상증자(악재)와 달리 호재로 분류할 수 있도록 구문 매칭 강화.
